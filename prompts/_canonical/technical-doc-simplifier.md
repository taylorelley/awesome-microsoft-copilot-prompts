## Context
Use this prompt when you need to simplify a technical document for non-technical audiences while maintaining accuracy.

## Prompt
```
Simplify this technical document for [AUDIENCE: executives/non-technical stakeholders/customers]. Maintain accuracy while: 1) Removing jargon or explaining it, 2) Using analogies for complex concepts, 3) Adding executive summary upfront, 4) Highlighting business impact vs. technical details, 5) Creating visual descriptions for data/processes, 6) Adding 'Why this matters' sections. Output in [FORMAT: summary/full rewrite/annotated version].
```
